Java重载VS重写

发表于 2021-08-14 更新于 2021-12-07 分类于 java

希望这篇文章能够帮助各位重载和重写傻傻分不清的朋友！

重写（Override）

重写一般是对父类的方法进行重写，要求

  • 返回值和形参都不能改变！
  • 返回值类型可以不相同，但必须是父类返回值的派生类（java 7↑）
  • 重写的方法能够抛出任何非强制性异常，但当父类方法本身也声明抛出的异常的时候，子类异常只能是父类异常的子类
  • 不能缩窄访问权限（父类方法是public，子类重写方法就不能为protected）
  • 声明为final的方法不能被重写
  • 声明为static的方法不能被重写，但是能够被再次声明
  • 子类和父类在同一个包中，那么子类可以重写父类所有方法，除了声明为 private 和 final 的方法。
  • 子类和父类不在同一个包中，那么子类只能够重写父类的声明为 public 和 protected 的非 final 方法。
  • 构造方法不能被重写

重载（Overload）

  • 被重载的方法必须改变参数列表(参数个数或类型或顺序不一样)；
  • 被重载的方法可以改变返回类型；
  • 被重载的方法可以改变访问修饰符；
  • 被重载的方法可以声明新的或更广的检查异常；
  • 方法能够在同一个类中或者在一个子类中被重载。
  • 无法以返回值类型作为重载函数的区分标准。

简单地讲：

重写：返回值和参数列表都不能改变！

重载：参数列表必须改变！

参数列表相同，返回值不同既不属于重载也不属于重写！！！
